Navigation Test

Check these things:

  1. Your current user role: Look at the top-right corner of the M365 AgentOps page. It should show your name and role (Admin or Super Admin). If it shows "user" or "manager", that's why you can't see User Investigation.
  2. Hard refresh the page: Press: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac) Wait for page to fully load.
  3. Check sidebar in Administration section: Look for these items in order:
    • Dashboard
    • Requests
    • Security
    • TenantGuard
    • User Investigation ← SHOULD BE HERE
    • Zero Trust
    • Privileged Accounts
  4. If still not visible, open browser console (F12) and paste this: 
    // Check user role
console.log('Current user role:', state.currentUser?.role)

// Check if user-investigation is in nav access
console.log('Nav access:', state.currentUser?.navAccess)

// Check if page div exists
console.log('Page div exists:', !!document.getElementById('page-user-investigation'))

// Check nav item exists
console.log('Nav item exists:', !!document.getElementById('n-user-investigation'))

Most Common Issue:

User role is not Admin or Super Admin
The page only appears for Admin and Super Admin users.
If you need access, your account needs to be added to an admin role.
